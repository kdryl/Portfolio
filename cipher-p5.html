Project Challenge

Build a complete cryptography tool that could encrypt and decrypt Caesar ciphers, handle various input methods, do statistical analysis, and demonstrate automated cryptanalysis techniques. This was my first substantial programming project.

Requirements

Support encryption/decryption with user-defined shifts, analyze character frequencies, handle keyboard and file input, implement brute-force decryption with dictionary validation, and include robust error handling.

Implementation

Core Algorithm

ASCII manipulation using ord() and chr() with modular arithmetic for character shifting

Frequency Analysis

Character frequency counting using collections.Counter for cryptanalysis support

Dictionary Validation

Set operations for fast word matching during automated decryption attempts

File Processing

Multi-line input handling with sys.stdin.readlines() and error management

Encryption Function

The core encryption processes each character individually, using modular arithmetic to shift letters within the alphabet while keeping non-alphabetic characters unchanged. This ensures compatibility with mixed text while maintaining classic Caesar cipher behavior.

Caesar Cipher Implementation

def encrypt_message(message, shift):
    coded = ''
    for letter in message:
        if ord(letter) < 65 or ord(letter) > 90:
            coded += letter
        else:
            coded += chr((ord(letter) + shift - 65) % 26 + 65)
    return coded

Advanced Features

Automated Cryptanalysis

Brute-force attack that tests all 26 possible shifts and validates results against English word dictionaries using set operations for fast candidate evaluation.

Statistical Analysis

Character frequency counting and pattern identification supporting manual cryptanalysis techniques and educational demonstration of cipher weaknesses.

Dictionary-Based Validation

The automated decryption tests each of the 26 possible shift values, comparing results against a dictionary of valid English words. Set operations provide O(1) lookup performance, enabling rapid evaluation and automatic identification of the correct shift.

User Interface

Interactive command-line interface with error handling loops for robust user experience. The system handles invalid inputs gracefully, provides clear feedback, and supports both single-use and repeated operations within the same session.
